# II. METHODS

## A. Study Design

This work is an original cross-sectional survey study on psychiatric medication use and public acceptance in Iraq. The analysis uses a unified cleaned dataset and reports prespecified main analyses separately from exploratory analyses.

## B. Participants and Analytical Samples

The overall dataset included 877 respondents. Because model variables had missingness across items, complete-case samples differed by model. The primary hierarchical logistic models used 647 complete cases, the multinomial model used 837 complete cases, and the sensitivity model with additional proximal belief items used 406 complete cases.

## C. Variables and Outcome Definitions

The primary outcome was willingness to recommend psychiatric medications, modeled in two ways. First, a binary outcome compared recommendation "Yes" versus "No". Second, a multinomial outcome preserved three response levels (No, Yes, Not sure) to retain hesitation as an analytically distinct state.

Predictor blocks included demographic variables (age, gender, education, marital status), prior use exposure, core belief items (Q11, Q12, Q13), and fear-related indicators. Q11 captured agreement that doctors prescribe psychiatric medications more than necessary. Q12 captured dependence concerns. Q13 captured beliefs that modern psychiatric medications are safer than older ones.

## D. Statistical Analysis Framework

The main publication-focused analysis followed two complementary model families. Hierarchical block logistic regression was used to evaluate incremental explanatory value from demographic factors, then prior use, then belief and fear variables. Model fit was summarized using McFadden pseudo R² and likelihood-ratio test p-values.

Multinomial logistic regression was then used to preserve the "Not sure" category rather than collapsing it, with relative risk ratios and confidence intervals reported for non-reference equations.

A sensitivity model added proximal belief variables (Q6 and Q7) and was reported separately because of conceptual proximity to recommendation outcome content.

## E. Exploratory Analyses

Exploratory analyses included user-versus-non-user comparisons on core belief items (Q11–Q13) and exploratory profile analysis using K-means clustering on standardized belief items. These analyses were labeled exploratory and interpreted as hypothesis-generating rather than confirmatory.

## F. Data Handling and Reporting Principles

Analyses were run on cleaned survey data with complete-case estimation per model. Reporting explicitly separates main/clean analyses from exploratory outputs to avoid overinterpretation and to preserve inferential clarity.
